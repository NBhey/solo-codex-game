# Аудит репозитория: чего сейчас не хватает проекту

Ниже — практичный список пробелов, которые стоит закрыть, чтобы шаблон превратился в устойчивый продакшен-проект.

## 1) Инженерные практики и качество кода

1. **Нет автоматических тестов** (unit/integration/e2e): в проекте отсутствуют тестовые файлы и тестовый раннер.
2. **Нет линтинга и форматтера** (ESLint/Prettier) в `package.json`, значит нет автоматической проверки стиля и потенциальных ошибок.
3. **Нет CI/CD пайплайна** (например, GitHub Actions) для обязательных проверок `typecheck/build` на каждый PR.
4. **Нет проверки бандла по размеру** — текущий build уже предупреждает о крупном чанке >500kB.

## 2) Геймдизайн и игровой контент

1. **Очень короткая core-loop**: по сути один режим «набей 10 фрагов», без прогрессии между сессиями.
2. **Нет реальной системы наград/мета-прогрессии** (перки, апгрейды, валюты, unlock’и).
3. **Нет разнообразия врагов/паттернов**: один базовый тип поведения (chase+shoot).
4. **Нет боссов, волн, событий, модификаторов раунда** — низкая реиграбельность.
5. **Нет полноценного обучения (tutorial/onboarding)**, особенно для тач-управления.

## 3) UX/UI и доступность

1. **Тумблер звука есть, но реального звука/музыки нет** (по коду не используются аудио-ассеты и audio API).
2. **Нет паузы во время матча** (кнопки/сцены паузы).
3. **Смешение языков в UI** (английский + русские строки) без i18n-слоя.
4. **Нет accessibility-настроек** (контрастные темы, масштаб UI, альтернативные схемы ввода).

## 4) Интеграция платформы и наблюдаемость

1. **Есть только базовая интеграция SDK**, но нет расширенной аналитики событий (победа/поражение/retention/funnel).
2. **Нет error-reporting/telemetry** (Sentry/LogRocket/свой endpoint) для прод-сбоев.
3. **Rewarded API обернут, но почти не используется в игровом цикле** (потенциал монетизации/второй шанс не раскрыт).

## 5) Производительность и технический долг

1. **Большой JS-бандл** (warning от Vite): стоит добавить code splitting/manualChunks.
2. **Нет lazy-loading для тяжелых частей** (сцены/сервисы грузятся сразу).
3. **Нет профилирования FPS/памяти** и целевых performance-бюджетов.

## 6) Операционная готовность

1. **Нет контейнеризации/инструкций деплоя** (Dockerfile, скрипты окружений), кроме базового README.
2. **Нет формализованного релизного процесса** (версии, changelog, release notes).
3. **Нет SECURITY/CONTRIBUTING/CODE_OF_CONDUCT** для масштабирования командной разработки.
4. **Нет LICENSE-файла** (юридически важно перед публичным распространением).

---

## Приоритетный roadmap (что делать первым)

1. **Надежность:** добавить ESLint + Prettier + Vitest + базовый CI (`typecheck`, `build`, тесты).
2. **Продукт:** внедрить звук/музыку и экран паузы, затем tutorial и 1-2 новых паттерна врагов.
3. **Монетизация:** встроить rewarded ads в понятный сценарий (например, revive 1 раз за ран).
4. **Производительность:** распилить бандл на чанки и проверить размер после оптимизаций.
5. **Операционка:** добавить LICENSE, CONTRIBUTING, SECURITY и шаблон релизов.
